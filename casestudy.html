Case Study for

NiliFlix - Angular Project




Overview

Using Angular, build the client-side for an application called NiliFlix that allows authorized users to access a list of movies. Users also can create an account, update their information, and add/remove a movie to their favorite movie list.

Purpose & Context

NiliFlix was a personal project that I built as a part of my web development course at CareerFoundry to practice with Angular and TypeScript.

Objective

The goal was to build a single-page, responsive movie app with Angular. The problem that I wanted to solve was to build an alternative client-side for the Niliflix application that allows users to use the app whenever they want and on any device and have an equivalent experience.



Approach

  1. Server-Side

    The server-side includes the server, business logic, and business layers of the application. MongoDB is a non-relational document model, which was the best option for my movie app project. I picked it for my database and used JavaScript, Node.js, and Express to interact with it. My well-designed REST API can be accessed via HTTP methods like GET,POST, PUT, and Delete. Similar methods, CRUD, were used to retrieve data from the database and store the data in a non-relational way. Postman is an excellent choice for testing the REST API manually. That’s why I chose it to simulate server-request and the result.

    (Click here to see the endpoints)


  2. Client-Side

    I started to build the user interface for making requests and receiving responses from the server-side. It is a single-page, responsive application developed with Angular and Angular Material. It provides several interface views, including the movies’ view, profile view, edit profile view, favorite movie view, login view, and registration. In detail, user stories will be:

    • Users can register and make a new account through the welcome view.
    • Users can log in to their accounts.
    • Authorized users can see a list of movies.
    • Users can click on a movie and see the description of the movie, Actors, Director, Genre, Rate, and Release year.
    • Users can add/remove a movie from their favorite list.
    • Users can edit /update their information.
    • Users can deregister.




Challenges

Working on this project was fun and challenging at the same time. TypeScript and Angular were new to me and different from React and React-Redux, which I had more experience in. I spent some time understanding how TypeScript and Angular work and getting the desired results. I overcame this challenge by studying some online resources and having a couple of calls with my great mentor.

Working on this project took some days longer than usual but helped me understand this framework and programming language better, and it turned out to be a fun experience.
